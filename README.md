⚙️ Принцип работы анализаторов
Скрипты разделены по модулям (module1_check.sh и module2_check.sh) и работают по следующему алгоритму:

1. Проверка Linux-машин (ISP, HQ-SRV, HQ-CLI, BR-SRV)
Для взаимодействия с Linux-системами используется встроенный механизм Proxmox — QEMU Guest Agent (qm guest exec).

Это позволяет выполнять команды внутри ВМ «без шума», не требуя настройки SSH-ключей или ввода паролей.

Скрипт отправляет команду (например, cat /etc/net/ifaces/enp7s1/ipv4address или systemctl is-active nginx), получает JSON-ответ, очищает его и сверяет результат с ожидаемым.

2. Проверка маршрутизаторов EcoRouter (HQ-RTR, BR-RTR)
Поскольку на EcoRouter нет QEMU Guest Agent, скрипт использует утилиту expect для взаимодействия через виртуальный последовательный порт (qm terminal):

Скрипт автоматически "пробивается" через консоль авторизации, используя умный перебор паролей (массив ROUTER_CREDS).

После успешного входа в привилегированный режим, скрипт выполняет точечные команды: show ip interface brief, show users localdb, show ip ospf, show running-config.

Вывод этих команд очищается от спецсимволов терминала и сохраняется в текстовые логи в директории /root/routereco/ (или /root/module2_check/).

Встроенные парсеры (на базе awk и grep) анализируют эти логи, находя конкретные IP-адреса, статусы интерфейсов (например, up), OSPF Router-ID и правила NAT.

🛠 Установка и настройка
Требования
Убедитесь, что на вашем хосте Proxmox установлена утилита expect. Если её нет, установите:

Bash
apt-get update && apt-get install expect -y
Настройка под ваш стенд
Перед запуском скриптов обязательно откройте их в любом текстовом редакторе (например, nano module1_check.sh) и актуализируйте блок переменных в самом начале файла:

Bash
# ============================================================================
# [ НАСТРОЙКИ ТОПОЛОГИИ И IP-АДРЕСОВ ] - МЕНЯЙТЕ ЗНАЧЕНИЯ ЗДЕСЬ!
# ============================================================================

# Укажите реальные ID ваших виртуальных машин в Proxmox
VM_ISP=13901
VM_HQ_RTR=13902
VM_HQ_SRV=13903
# ... и так далее
Important
Скрипт использует глобальные переменные (например, IP_HQ_RTR_VL100="192.168.100.1/27"). Если в задании изменятся IP-адреса, достаточно поменять их в этом блоке — вся логика проверок обновится автоматически.

🚀 Использование
Сделайте скачанные скрипты исполняемыми:

Bash
chmod +x module1_check.sh
chmod +x module2_check.sh
Запуск проверок Модуля 1
Модуль 1 проверяет базовую адресацию (А1Д1, А2Д1), OSPF (АЗД1), DHCP, учетные записи (Б1Д1, Б2Д1), DNS, туннели GRE (В1Д1, В2Д1) и подинтерфейсы.

Bash
./module1_check.sh
Запуск проверок Модуля 2
Модуль 2 фокусируется на сервисах (Связка Nginx Reverse Proxy, RAID, NFS, Samba AD DC, Ansible, Docker) и правилах статического NAT на EcoRouter. Скрипт поддерживает выборочный запуск:

Bash
# Запустить полную проверку (и Linux, и Роутеры)
./module2_check.sh full

# Проверить только Linux-сервисы (быстро)
./module2_check.sh linux

# Проверить только конфигурации роутеров (NAT, NTP и т.д.)
./module2_check.sh routers
📊 Интерпретация результатов
В процессе работы скрипт выводит лог подключения и сбора данных (синим цветом). После сбора формируется итоговая таблица:

Скрипт группирует проверки по критериям из ведомости: РАЗДЕЛ А, РАЗДЕЛ Б, РАЗДЕЛ В и т.д.

Каждая проверка показывает: проверяемый узел, код критерия, требуемое значение и фактическое значение, найденное в системе.

Зеленый статус совпадает — настройка выполнена верно.

Красный статус не совпадает — настройка отсутствует ([НЕ НАСТРОЕНО / ПУСТО]) или выполнена с ошибкой.

Желтый статус ВНИМАНИЕ — ручные критерии (например, Д1Д1 - проверка оформления отчета по ГОСТ), требующие визуального осмотра экспертом.
